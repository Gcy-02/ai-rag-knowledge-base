# 项目讲解笔记

这份笔记主要给自己面试前复习用，不是正式产品文档。

## 我为什么这么做

这个项目的目标不是一次性做成企业级知识库，而是先把 RAG 的主流程跑通：

```text
上传资料 -> 切块 -> Embedding -> ChromaDB -> Top-K 检索 -> GPT 回答 -> 来源引用
```

我先没有做登录、权限、Docker、OCR 和复杂文档管理。原因是 V1 最重要的是证明 RAG 链路真的能跑，而且面试官打开 GitHub 能直接看到页面和截图。

## 我能讲清楚的点

- `company.txt` 是内置知识库，方便最小版本运行。
- PDF 上传后会解析文本，再切成知识块。
- OpenAI Embedding 负责把问题和知识块都转成向量。
- ChromaDB 负责做相似度检索。
- `/ask` 返回答案、引用来源和 Top-K 召回结果。
- 页面上展示文档数、知识块数和 Top-K，方便看到检索过程。

## 当前边界

- PDF 只处理文本型 PDF，不做 OCR。
- 上传文件没有做删除和后台管理。
- Demo Mode 只是为了展示页面，不等于真实 Embedding 检索。
- ChromaDB 存在本地目录里，适合 Demo，不是生产部署方案。

## 面试时可以怎么说

我会说这个项目先做的是 RAG 的可验证主流程。后续如果继续升级，我会优先做两个点：第一是文档管理，比如删除、重新索引；第二是引用定位更精确，比如标到 PDF 页码甚至段落。
